API, Traffic Data, Dashboard

내비게이션 앱 3사 경로 추천 알고리즘 비교 분석

사용자가 느끼는 경로 추천의 아쉬움을 리뷰 데이터로 확인하고, 서울의 교통 병목 지점과 앱별 경로 API 데이터를 결합해 어떤 상황에서 어떤 앱이 더 적합한지 분석했습니다.

Rest APICrontabAWS EC2 TableauKcBERTTraffic Data

역할

프로젝트 리더로 아이디어 구체화, 데이터 수집, 리뷰·교통·경로 데이터 분석, 비교 대시보드 제작을 맡았습니다. 단순히 API를 호출하는 것보다, 어느 시간대와 어떤 경로 조건에서 비교해야 의미가 있는지 정하는 데 집중했습니다.

분석 과정

리뷰 데이터 분석

Google Play Scraper와 App Store Scraper로 네이버지도, 카카오맵, 티맵 리뷰를 각각 약 2만 건씩 수집하고, KcBERT 기반 감정분석을 적용했습니다.

교통 데이터 분석

서울 열린데이터 광장과 한국도로공사 데이터를 활용해 출퇴근과 주말 시간대의 병목 구간을 확인하고 출발지와 도착지 후보를 정했습니다.

경로 API 수집

AWS EC2에서 Crontab으로 Naver Cloud Platform, SK open API, Kakao 지도 API 호출을 자동화하고 시간대별 추천 경로를 수집했습니다.

주요 결과

  • 주말 장거리 기준으로 예상 시간은 네이버, 비용은 티맵, 거리는 카카오가 각각 강점을 보였습니다.
  • 티맵은 경유지가 많은 경로에서 안정적인 성능을 보였습니다.
  • 네이버는 유류비 예측이 포함되어 비용 효율 비교에 강점이 있었습니다.
  • 카카오는 최단거리 옵션에서 강점이 있었지만, 소요시간 측면에서는 경쟁력이 약하게 나타났습니다.
