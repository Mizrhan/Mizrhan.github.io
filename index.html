데이터의 기준을 맞추고, 결과가 왜 그렇게 나왔는지 끝까지 확인합니다.

통계와 실험계획을 바탕으로 데이터를 정리하고, 지표와 모델 결과를 실제 판단에 쓸 수 있는 형태로 만듭니다. SQL 기반 KPI 분석, API와 Crontab을 활용한 데이터 수집 자동화, 시정 예측 모델 고도화처럼 데이터 수집부터 전처리, 모델링, 해석, 리포트와 대시보드 전달까지 이어지는 흐름을 경험했습니다.

강점

포트폴리오의 중심은 도구 나열이 아니라, 현실 데이터의 문제를 어떤 기준으로 다루고 설명했는지입니다.

3회

공공 데이터 분석 수상

날씨 빅데이터 콘테스트, 연구데이터 활용 경진대회, 병무청 등 합동 데이터 분석 및 아이디어 공모전에서 수상했습니다.

13%

시정 예측 성능 개선

기존 예측 방식 대비 6개 지역 평균 CSI를 0.3795에서 0.4286으로 개선하고, 성능 변동 원인을 분석했습니다.

API

수집 자동화 경험

내비게이션 앱 3사의 경로 API를 AWS EC2와 Crontab으로 주기적으로 호출해 시간대별 비교 데이터를 수집했습니다.

왜

결과 원인 분석

SHAP, PDP, Wasserstein Distance를 활용해 모델의 성능 차이와 분포 변화의 원인을 확인했습니다.

대표 프로젝트

데이터 분석가 지원 기준으로 먼저 보여줄 프로젝트입니다. 연구형 모델링, API 수집 자동화, SQL KPI 분석, 고객 분석을 균형 있게 배치했습니다.
